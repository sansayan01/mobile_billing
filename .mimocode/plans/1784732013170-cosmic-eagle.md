# Full Bill Edit — Implementation Plan

## Goal
Edit button dabane pe pura receipt edit kar sake — items (qty/price), add/remove items, payment method — with proper stock management.

## Current State
- Edit dialog sirf 3 fields allow karta hai: `customer_name`, `customer_phone`, `discount`
- `bill_items` table mein sirf READ hota hai report feature mein — koi insert/update/delete nahi
- Stock adjustment billing creation ke time hota hai (billing_bloc), koi reusable method nahi

## Scope

### Editable Fields
| Field | Current | After |
|---|---|---|
| Customer Name | Yes | Yes |
| Customer Phone | Yes | Yes |
| Discount | Yes | Yes |
| Payment Method | No | Yes (dropdown: upi/cash/card) |
| Item Quantity | No | Yes |
| Item Price | No | Yes |
| Remove Item | No | Yes |
| Add New Item | No | Yes (product search dialog) |
| Grand Total | Auto-calc | Auto-calc |

### Stock Rules
- **Qty increase** → stock kam karo (deduct diff)
- **Qty decrease** → stock badhao (restore diff)
- **Item remove** → stock poori quantity restore
- **New item add** → stock deduct karo
- **Price change** → sirf bill_items update, stock pe koi effect nahi

**File:** `report_repository.dart`, `report_repository_impl.dart`
- `updateBill` mein `List<BillItem>? items` parameter add karo
- Full update logic with stock management implement karo:
  - Fetch existing items from `bill_items` table
  - Diff computation: compare existing vs new items by `productId`
  - For removed items: `DELETE` bill_item + `UPDATE products.stock` (+qty) + `INSERT inventory_log`
  - For modified items: `UPDATE bill_items` (qty/price) + `UPDATE products.stock` (diff) + `INSERT inventory_log`
  - For added items: `INSERT bill_item` + `UPDATE products.stock` (-qty) + `INSERT inventory_log`
  - After all item changes: recalculate `total_amount`, `grand_total`, `item_count` on `bills` table

## Edge Cases to Handle

1. **Empty bill after edits** → Allow save? Or require at least 1 item? → Require at least 1 item
2. **Stock going negative** → Before save, validate all items have sufficient stock. Show error if not.
3. **Discount recalculation** → When items change, recalculate grandTotal = totalAmount - discount
4. **Same product added twice** → If user adds a product that already exists in bill, increment its qty instead of adding duplicate
5. **Network failure mid-save** → Best effort — some items may update, others not. Re-fetch bill on any error.
